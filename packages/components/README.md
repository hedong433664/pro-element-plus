# @pro-element-plus/components

## 包说明

`components` 是组件源码层，负责维护每个组件的实现、对外入口与样式源码。这里是整个组件库最核心的业务实现目录。

## 当前工程结构

```text
components
├─ col/                 ProCol 组件
├─ header/              ProHeader 组件
├─ table/               ProTable 组件
├─ styles/              组件公共样式与总样式源码
├─ index.ts             组件聚合导出
└─ package.json         当前子包依赖说明
```

每个组件目录通常包含：

```text
<component>
├─ src/                 组件源码与类型
├─ style/               组件样式依赖入口
└─ index.ts             组件对外导出入口
```

## 作用

- 编写与维护组件逻辑
- 维护组件级类型导出
- 维护组件级样式依赖清单

## 开发方式

通常不会单独运行本包，而是配合以下子包联调：

- 在 `play` 中预览组件行为
- 在 `docs` 中维护正式示例
- 通过 `core` 的构建脚本完成发包产物生成

## 开发规范

- 组件目录下的 `index.ts` 只负责组件导出与安装增强，不堆放实现逻辑。
- 类型尽量收敛在 `src/*.ts` 中，方便组件与文档共享。
- `style/index.ts` 与 `style/css.ts` 是样式依赖清单的唯一来源，新增依赖时必须同步维护。
- 公共样式放在 `styles/`，组件局部样式优先按组件维度组织。
- 新增组件时，至少补齐：源码、类型、样式入口、聚合导出。

## 协作建议

- 如果改动会影响发包结构、导出路径或构建行为，需要同步检查 `core`。
- 如果改动影响示例或说明，需要同步更新 `play` / `docs`。
